1. The Attention Mechanism

The Transformer processes a sequence of tokens through a series of well-defined steps. Understanding each step is key to grasping why Transformers are so powerful.

Step 1 — Input Sequence

Start with $N$ tokens:

$$\text{Input} = [w_1, w_2, \ldots, w_N]$$

Step 2 — Word Embeddings

Each token is mapped to a dense vector in $\mathbb{R}^d$:

$$\{c_1, c_2, \ldots, c_N\}, \quad c_i \in \mathbb{R}^d$$

Step 3 — Positional Embeddings

Transformers have no inherent notion of word order. Positional encodings fix this by adding position-dependent vectors:

$$\{p_1, p_2, \ldots, p_N\}, \quad p_i \in \mathbb{R}^d$$

Step 4 — Combined Input

Word and positional embeddings are summed element-wise:

$$\{c_1+p_1,\; c_2+p_2,\; \ldots,\; c_N+p_N\}$$

Step 5 — Attention Network

For each position $i$, the attention mechanism gathers a weighted combination of all positions' values:

$$\tilde{c}_i = \sum_{j=1}^N \text{Attention}(i,j) \cdot (W_V (c_j+p_j))$$

Step 6 — Skip Connection (Residual)

The original input is added back — this residual connection ensures gradient flow during training and preserves the original representation:

$$(c_i+p_i) + \tilde{c}_i$$

Step 7 — Add & Normalize

Layer normalization is applied after the residual add:

$$\text{Output} = \Big\{n(c_1+p_1+\tilde{c}_1),\; n(c_2+p_2+\tilde{c}_2),\; \ldots,\; n(c_N+p_N+\tilde{c}_N)\Big\}$$

Summary: Each token attends to all other tokens in the sequence, producing a context-enriched representation. The skip connection preserves the original identity, while layer norm stabilizes training. The result: contextualized representations ready for the next layer.

2. Multi-Head Self-Attention

Instead of a single attention function, multi-head attention runs $h$ attention operations in parallel, each with its own learned projections:

$$\text{Multi-Headed Output} = \text{Concat}(\text{output}_1, \ldots, \text{output}_h)\, W^O$$ $$\text{output}_i = \text{Attention}(W_i^K K,\; W_i^V V,\; W_i^Q Q)$$

In self-attention, the queries, keys, and values all come from the same input; $K$, $Q$, and $V$ are identical before projection. The $h$ projection matrices allow each head to attend based on different aspects of the word vectors — different topics, writing styles, or syntactic roles.

Interaction with Positional Encodings

For word $n$, the combined embedding is $w_n = e_n + p_n$. By linearity:

$$W_i^Q w_n = W_i^Q e_n + W_i^Q p_n, \quad W_i^K w_n = W_i^K e_n + W_i^K p_n, \quad W_i^V w_n = W_i^V e_n + W_i^V p_n$$

The sinusoidal positional encoding is defined as:

$$PE_{(pos,\, 2i)} = \sin\!\left(\frac{pos}{10000^{2i/d}}\right), \qquad PE_{(pos,\, 2i+1)} = \cos\!\left(\frac{pos}{10000^{2i/d}}\right)$$

Each head can thus learn to exploit positional information in different ways:

  1. Positional frequencies — components oscillate at different rates with position.
  2. Proximity scaling — head matrices can emphasize components of $p_n$ that correspond to particular scales of closeness.
  3. Content emphasis — head matrices can also emphasize components of $e_n$, highlighting particular topics or writing styles.

Example: Positional-Only Head

One head may learn $W_i^Q e_n \approx \mathbf{0}$ and $W_i^K e_n \approx \mathbf{0}$, so that attention weights are determined almost purely by positional proximity — directing attention to words near the next word to be predicted.

Example: Content-Only Values Head

Another head may combine positional keys/queries with content-rich values: $W_i^V p_n \approx \mathbf{0}$ while $W_i^V e_n \neq \mathbf{0}$, so the gathered information captures semantic meaning of nearby words.

3. Next Word Prediction & Training

Autoregressive Generation

In autoregressive models like GPT, attention is masked: each token attends only to itself and previous tokens — never to future ones. After $L$ layers, the network produces output vectors:

$$\{f_1^{(L)}, f_2^{(L)}, \ldots, f_N^{(L)}\}$$

To predict the $(N+1)$-th token, we use the last position's representation $f_N^{(L)} \in \mathbb{R}^d$, pass it through a learned projection $M \in \mathbb{R}^{d \times d}$, and compute $u = M f_N^{(L)}$.

The probability of token $v$ being the next word is then:

$$p(v \mid \text{prompt}) = \frac{\exp[w_v^T u]}{\displaystyle\sum_{v'=1}^{V} \exp[w_{v'}^T u]}$$

where $\{w_v\}_{v=1}^V$ are the learned vocabulary embedding vectors. The last token sees information from all previous tokens through stacks of masked attention layers.

Training via Cross-Entropy Loss

Given a corpus of $J$ prompts $\{\text{prompt}^{(j)}\}_{j=1}^J$, each with a true next token $w_{N+1}^{(j)}$, all Transformer parameters $\theta$ are trained by minimizing the cross-entropy loss:

$$\hat{\theta} = \arg\min_{\theta}\left[-\sum_{j=1}^{J} \log\, p_\theta\!\left(v = w_{N+1}^{(j)} \;\Big|\; \text{prompt}^{(j)}\right)\right]$$

Intuition: The model is rewarded for assigning high probability to the true next token across all prompts. Over millions of examples, this objective forces the model to learn grammar, facts, reasoning patterns, and more.

4. The Feedforward Layer

After the attention sub-layer comes the position-wise feedforward network (FFN) — a critical component that adds nonlinearity and processing capacity to each Transformer block.

Mathematical Formulation

For each token vector $\mathbf{x} \in \mathbb{R}^{d_{\text{model}}}$:

$$\text{FFN}(\mathbf{x}) = W_2 \cdot \text{Activation}(W_1 \mathbf{x} + b_1) + b_2$$

where:

  • $W_1 \in \mathbb{R}^{d_{\text{ff}} \times d_{\text{model}}}$, $b_1 \in \mathbb{R}^{d_{\text{ff}}}$ — first (expansion) layer
  • $W_2 \in \mathbb{R}^{d_{\text{model}} \times d_{\text{ff}}}$, $b_2 \in \mathbb{R}^{d_{\text{model}}}$ — second (compression) layer
  • Activation: typically ReLU or GELU; $d_{\text{ff}} = 4 \times d_{\text{model}}$

Activation Functions

$$\text{ReLU}(x) = \max(0, x)$$ $$\text{GELU}(x) = x \cdot \Phi(x) \quad \text{where } \Phi(x) \text{ is the standard Gaussian CDF}$$

5. Encoder–Decoder Architecture

The full Transformer consists of two main components:

  • Encoder — Builds a contextual understanding of the input sequence.
  • Decoder — Generates the output sequence using the encoder's representation plus previously generated tokens.

Encoder

Given an input sequence $X \in \mathbb{R}^{T \times d}$, each encoder layer applies:

  1. Self-Attention — Every token attends to all others:
    $$\text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$ $$Q = XW_Q,\quad K = XW_K,\quad V = XW_V, \quad W_Q, W_K, W_V \in \mathbb{R}^{d_{\text{model}} \times d_k}$$
    Compactly: $\text{SelfAttention}(X) = \text{softmax}\!\left(\dfrac{(XW_Q)(XW_K)^T}{\sqrt{d_k}}\right)(XW_V)$
  2. Feedforward Network — Position-wise nonlinear transformation.

After several layers the encoder produces a memory of the input:

$$E = \text{Encoder}(X), \quad E \in \mathbb{R}^{T \times d}$$

Decoder

The decoder generates tokens autoregressively. Each decoder layer has:

  1. Masked Self-Attention — Each position attends only to past tokens.
  2. Cross-Attention — The key interaction between encoder and decoder:
    $$Q = \text{decoder states},\quad K, V = E$$
    The decoder queries the encoder memory to retrieve relevant source information.
  3. Feedforward Network — Further transforms the combined information.

Key Insight: Cross-attention eliminates the information bottleneck present in older RNN-based sequence-to-sequence models, where the entire source had to be compressed into a single fixed-size vector. The decoder can directly attend to any part of the source at every layer.

6. Encoder Details & Linear Attention Analysis

To build mathematical intuition, consider a simplified single-head encoder with shared parameters $W_Q, W_K, W_V, P \in \mathbb{R}^{d \times d}$ across all layers.

The Update Rule

The output at position $i$ from layer $l$, with skip connection:

$$e_i^{(l)} = e_i^{(l-1)} + P \sum_{j=1}^{N} \bigl(W_V e_j^{(l-1)}\bigr)\, w_{ij}^{(l-1)}$$

where the attention weight is a softmax over query-key similarities:

$$w_{ij}^{(l-1)} = \frac{\exp\!\left[\lambda\, (W_Q e_i^{(l-1)})^T (W_K e_j^{(l-1)})\right]} {\displaystyle\sum_{j'=1}^{N} \exp\!\left[\lambda\, (W_Q e_i^{(l-1)})^T (W_K e_{j'}^{(l-1)})\right]}$$
SymbolMeaningIntuition
$e_i^{(0)}$Input at position $i$Initial word representation (e.g., "bank" — river or money?)
$e_i^{(1)}$Output at position $i$Contextualized representation after seeing the whole sentence
$P$Projection matrix"How much should I revise my understanding based on context?"
$W_V e_j^{(0)}$Value of position $j$Semantic content of word $j$
$w_{ij}^{(0)}$Attention weightHow relevant is word $j$ to understanding word $i$?

Linear Attention Simplification

Setting $\Phi = W_Q^T W_K = I$ (identity) gives the linear attention update — a simplification that captures the core structure of attention:

$$e_i^{(l)} = e_i^{(l-1)} + P \sum_{j=1}^{N} (W_V e_j^{(l-1)})\, \underbrace{\bigl[(e_i^{(l-1)})^T e_j^{(l-1)}\bigr]}_{\text{linear attention}}$$

In matrix form, with $E_l = (e_1^{(l)}, \ldots, e_N^{(l)}) \in \mathbb{R}^{d \times N}$:

$$E_l = E_{l-1} + P \underbrace{W_V E_{l-1}}_{\text{Values}} \underbrace{[E_{l-1}^T E_{l-1}]}_{\text{Attention weights}}$$

7. In-Context Learning & Few-Shot Meta-Learning

The linear attention framework above reveals a surprising and beautiful connection between Transformer forward passes and gradient descent — explaining how LLMs can learn from a handful of examples in their context window.

The Setup

Consider contextual data $C = \{(x_i, y_i)\}_{i=1}^N$ where $x_i \in \mathbb{R}^d$, $y_i \in \mathbb{R}$, and an unknown function $y_i = f(x_i)$. Given $C$, predict $y_{N+1} = f(x_{N+1})$ for a new query. This is few-shot learning (also called meta-learning): Learning to learn — a model trained on $M$ such tasks should adapt to a new task at inference time, without changing any parameters.

Hypothesis: Attention Implements Gradient Descent

For the linear model $f_\theta(x) = w^T x$ and the squared loss:

$$\mathcal{L}(w) = \frac{1}{2}\sum_{i=1}^N (y_i - w^T x_i)^2$$

Gradient descent produces the update:

$$w_{k+1} = w_k + \alpha \sum_{i=1}^N (y_i - w_k^T x_i)\, x_i$$

In function space, this is:

$$f_{\theta_{k+1}}(x_j) = f_{\theta_k}(x_j) + \alpha \sum_{i=1}^N \bigl[y_i - f_{\theta_k}(x_i)\bigr]\, x_i^T x_j$$

Encoding the Context

Encode each context point as a $(d+1)$-dimensional vector:

$$e_{1,0} = \begin{pmatrix} x_1 \\ y_1 \end{pmatrix}, \quad e_{2,0} = \begin{pmatrix} x_2 \\ y_2 \end{pmatrix}, \quad \ldots, \quad e_{N+1,0} = \begin{pmatrix} x_{N+1} \\ 0 \end{pmatrix}$$

Design the parameter matrices as:

$$\Phi = \begin{pmatrix} I_{d} & 0 \\ 0 & 0 \end{pmatrix}, \quad W_V = \begin{pmatrix} 0_{d\times d} & 0 \\ 0 & 1 \end{pmatrix}, \quad P = \begin{pmatrix} 0_{d\times d} & 0 \\ 0 & -\alpha \end{pmatrix}$$

What Happens at Each Layer

After layer $k$, position $j$ holds:

$$e_{j,k} = \begin{pmatrix} x_j \\[4pt] y_j - \displaystyle\sum_{k'=1}^k \Delta f_{\theta_{k'}}(x_j) \end{pmatrix}$$

The bottom component tracks the residual error of the model at step $k$. The linear attention output at layer $k+1$ contributes exactly:

$$\begin{pmatrix} 0 \\ -\Delta f_{\theta_{k+1}}(x_j) \end{pmatrix}$$

Via the skip connection, this updates $e_{j,k}$ to $e_{j,k+1}$ — incrementally tracking the cumulative gradient descent update.

The Punchline: A Transformer with linear attention, with properly designed parameters, performs gradient descent in its forward pass. Each layer $=$ one GD step. The skip connections accumulate the updates. No parameter changes required at inference time — just the forward pass itself is learning from context.

8. Extending to Nonlinear Functions: The Kernel Perspective

The linear model analysis extends naturally to nonlinear functions via kernel methods — bridging kernel machines and Transformer attention.

Feature Mapping & Kernel Trick

Map inputs to a (possibly infinite-dimensional) feature space $\phi(x) \in \mathbb{R}^{d'}$:

$$f_\theta(x) = w^T \phi(x) + b$$

The kernel trick avoids computing $\phi(x)$ explicitly; work with inner products instead:

$$\kappa(x_i, x_j) = \phi(x_i)^T \phi(x_j)$$

Common Kernel Functions

KernelFormula
Linear$\kappa(x_i, x_j) = x_i^T x_j$
RBF (Gaussian)$\kappa(x_i, x_j) = \exp\!\left(-\|x_i - x_j\|_2^2 / \sigma^2\right)$
Laplacian$\kappa(x_i, x_j) = \exp\!\left(-\|x_i - x_j\|_1 / \sigma^2\right)$
Polynomial$\kappa(x_i, x_j) = (\gamma x_i^T x_j + c)^d$
Cosine$\kappa(x_i, x_j) = x_i^T x_j / (\|x_i\|_2 \|x_j\|_2)$

Kernel Gradient Descent

GD in feature space produces the function-space update:

$$f_{w_{k+1}}(x_j) = f_{w_k}(x_j) + \alpha \sum_{i=1}^N \bigl[y_i - f_{w_k}(x_i)\bigr]\, \kappa(x_i, x_j)$$

This matches the linear attention update — with $\kappa(x_i, x_j)$ playing the role of the attention score. The entire kernel GD algorithm can be implemented as a Transformer forward pass!

Two-Head Attention for Bias Terms

When a bias $b$ is included in $f_\theta$, gradient descent produces two update terms (kernel + constant). These can be implemented with two attention heads:

  • Head 1 (Kernel head) — Computes RBF kernel similarity between positions.
    $$W_Q^{(1)} = W_K^{(1)} = \begin{pmatrix} I_d & 0 \\ 0 & 0 \end{pmatrix}, \quad \kappa(W_K^{(1)} e_i,\, W_Q^{(1)} e_j) = \exp\!\left(-\frac{\|x_i - x_j\|_2^2}{\sigma^2}\right)$$
  • Head 2 (Bias head) — Provides constant attention weight of 1 for the bias term.
    $$W_Q^{(2)} = W_K^{(2)} = 0_{(d+1)\times(d+1)}, \quad \kappa(\cdot,\cdot) = 1$$

Both heads share the same value and output matrices:

$$W_V^{(1)} = W_V^{(2)} = \begin{pmatrix} 0_{d\times d} & 0 \\ 0 & 1 \end{pmatrix}, \quad W_O^{(1)} = W_O^{(2)} = \begin{pmatrix} 0_{d\times d} & 0 \\ 0 & -\alpha \end{pmatrix}$$

Formal Equivalence: Linear Attention = Gradient Descent

With the encoding and matrix design from Section 6, we can verify step by step that the Transformer's forward pass over $L$ layers computes exactly $L$ steps of gradient descent:

$$f_{\theta_L}(x_{N+1}) = \sum_{k=1}^L \alpha \sum_{i=1}^N \bigl(y_i - f_{\theta_{k-1}}(x_i)\bigr)\, x_i^T x_{N+1}$$

After $L$ layers, the query position $N+1$ carries:

$$e_{N+1,L} = \begin{pmatrix} x_{N+1} \\ -\displaystyle\sum_{k'=1}^L \Delta f_{\theta_{k'}}(x_{N+1}) \end{pmatrix}$$

Taking the inner product with $(0_d, -1)^T$ extracts the final prediction $f_{\theta_L}(x_{N+1})$.

Key Insights from the Kernel Perspective:

  • Nonlinear attention arises naturally from different choices of kernel $\kappa$.
  • Multi-head attention can implement different components of the GD update (e.g., kernel term + bias term).
  • The skip connection is essential — it accumulates gradient descent updates across layers.
  • This framework provides a principled explanation of in-context learning: the Transformer literally performs GD on the prompt examples during its forward pass.
  • Softmax attention approximates a valid Mercer kernel, grounding the intuition in RKHS theory.
Transformers with kernel-based attention can perform gradient descent for nonlinear functions entirely within their forward pass — explaining their powerful few-shot learning capabilities without any parameter updates at inference time.
